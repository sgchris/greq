Fast, declarative API testing for builders

Greq is a Rust-powered HTTP testing tool. Define requests, dependencies, and assertions in readable .greq files. Run them locally or in CI to validate APIs quickly and reliably.

Why Greq

Declarative tests

Write HTTP requests and assertions in a single file. Clean structure with header/content/footer sections, easy to review and maintain.

Dependencies & placeholders

Chain requests and reuse values with $(dependency...) placeholders. Compose complex flows without glue code.

Rich conditions

Assert status, headers, and JSON paths. Detailed failure messages with actual vs expected help you debug faster.

Rust-performance

Built with Rust and async I/O. Run many tests in parallel with clear logs and compact, colored output.

Examples

Simple GET

project: simple-get
is-http: true

====

GET /status/200 HTTP/1.1
host: httpbin.org

====

status-code equals: 200

Dependency with placeholders

project: user-by-id
depends-on: 01-create-user.greq
is-http: true

====

GET /users/$(dependency.response-body.id) HTTP/1.1
host: example.org

====

status-code equals: 200
response-body contains: $(dependency.response-body.id)

See more in greq-examples.
